## What's New in This Fork

This fork adds research-backed improvements grounded in 9+ published papers on AI-assisted security auditing:

### 🛡️ Critic Meta-Agent (New)
An adversarial 9th agent that re-evaluates all findings from the 8 hacking agents. Scores each finding on correctness × severity × exploitability and eliminates false positives. Based on the [GPTLens](https://arxiv.org/abs/2310.01152) adversarial auditor-critic pattern and [iAudit](https://arxiv.org/abs/2403.16073) ranker-critic debate protocol.

### 🔬 PoC Generator Agent (New)
Automatically writes Foundry proof-of-concept tests for confirmed findings (confidence ≥ 80). Three-phase approach: Strategic Planner → Tactical Executor → Adaptive Refiner. Based on the [SAILOR](https://arxiv.org/abs/2604.06506) dual-loop pattern.

### 🔍 Static Analysis Pre-Pass (New)
Optional Slither/Aderyn integration that runs before LLM agents. Provides structured call graphs, state variable maps, and taint flows as pre-analysis context. Based on the [IRIS](https://arxiv.org/abs/2405.17238) LLM+static analysis pattern.

### 📊 Enhanced Evaluation System
- Per-category recall tracking (access-control, math, reentrancy, etc.)
- Precision + F1 score calculation
- False positive counting and classification
- Regression detection across runs
- Aggregate cross-benchmark reporting

### 🧬 20 New Attack Vectors (267-286)
Covering 2024-2026 exploit patterns:
- L2 sequencer downtime, EIP-1153 transient storage, EIP-4844 blob pruning
- Permit2 griefing, ERC-4337 validation manipulation, restaking cascades
- Uniswap V4 hook reentrancy, EIP-7702 delegation races
- Intent-based solver collusion, ERC-7579 module forgery
- Concentrated liquidity oracle manipulation, multi-collateral cherry-picking

### 📡 X-Ray Call-Graph Profiling (New)
Optional Slither-powered inter-procedural context extraction for X-Ray reports. Maps callers, callees, and shared state per function to enhance invariant synthesis and attack surface identification. Based on [CPRVul](https://arxiv.org/abs/2602.06751) context profiling.


```
update skills to latest version
```

### New Flags

| Flag | Description |
|------|-------------|
| `--poc` | Generate Foundry PoC tests for confirmed findings (auto-enabled for confidence ≥ 85) |
| `--no-critic` | Skip the Critic validation pass for faster runs |
| `--no-static` | Skip static analysis pre-pass even if Slither/Aderyn are installed |

### Optional Dependencies

For maximum coverage, install these (all optional — the skill works without them):

```bash
pip install slither-analyzer   # Static analysis pre-pass
cargo install aderyn            # Secondary static analysis
```

## Research References

This fork's improvements are grounded in published research:

| Paper | Key Contribution | Applied Where |
|-------|-----------------|---------------|
| [GPTLens](https://arxiv.org/abs/2310.01152) (Georgia Tech) | Adversarial auditor-critic architecture | Critic Agent |
| [iAudit](https://arxiv.org/abs/2403.16073) (MetaTrust/NTU) | Multi-prompt voting + ranker-critic debate | Critic Agent scoring |
| [IRIS](https://arxiv.org/abs/2405.17238) | LLM + static analysis hybrid | Static Analysis Pre-Pass |
| [Vul-RAG](https://arxiv.org/abs/2406.11147) | Knowledge-level RAG for vuln detection | Attack vector structure |
| [CPRVul](https://arxiv.org/abs/2602.06751) | Inter-procedural context profiling | X-Ray call-graph profiling |
| [SAILOR](https://arxiv.org/abs/2604.06506) | Symbolic execution guided by LLM | PoC Generator Agent |
| [FORGE](https://arxiv.org/abs/2506.18795) | Large-scale audit dataset construction | Eval methodology |
| [BugLens](https://arxiv.org/abs/2504.11711) | LLM post-refines static analysis | Static Analysis Pre-Pass |
